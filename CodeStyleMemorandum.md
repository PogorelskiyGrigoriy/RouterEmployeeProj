Меморандум по стилю кода (Code Style Guide)
1. Структура файла и экспорт
Именованный экспорт: Всегда используем стрелочные функции.
Почему: Четкое имя при авто-импорте, отсутствие «всплытия» (hoisting).
Константы: Статические данные (default values, схемы) выносятся вне тела компонента.
Типизация пропсов: Всегда используем interface Props в верхней части файла.
2. Организация импортов (Группировка)
Импорты идут от «внешнего мира» к «сердцу проекта»:
Библиотеки: React, Hook Form и др.
UI слой: Chakra UI и локальные UI-компоненты.
Иконки: React-icons.
Логика: Хуки, Store, API.
Типы: Инпуты/Аутпуты данных.
Utils: Конфиги и хелперы.
3. Правила React & TypeScript
Деструктуризация: Пропсы извлекаются прямо в аргументах функции.
События:
on[Event] — для имен в Props (входящие колбэки).
handle[Event] — для функций внутри компонента (реализация).
Interface vs Type: * interface — для объектов и пропсов.
type — для простых типов (ID, Role) и объединений (Union).
4. Работа с Chakra UI (Styling)
Zero Inline Styles: Никаких style={{...}}. Только системные пропсы (mt, bg).
Вычисляемые стили: Сложные условия для оформления выносятся в переменные перед return.
